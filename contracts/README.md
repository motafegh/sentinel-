# M5 — Solidity Contracts

Foundry project for all on-chain SENTINEL components. Three contracts implement the trust model: an ERC-20 staking token for economic collateral, a UUPS upgradeable audit registry, and a ZK verifier (auto-generated by EZKL).

---

## Contract Architecture

```
SentinelToken (ERC-20 + staking)
  stake(amount)         → lock SNTL, become audit-eligible agent
  unstake(amount)       → return locked tokens
  slash(agent, amount)  → burn dishonest agent's stake (onlyOwner)
  stakedBalance(agent)  → view staked amount
  MIN_STAKE = 1 000 SNTL (1e18 decimals)

AuditRegistry (UUPS upgradeable)
  submitAudit(contractAddress, scoreFieldElement, proof, publicSignals)
    Guard 1 — economic:      stakedBalance(msg.sender) >= MIN_STAKE
    Guard 2 — cryptographic: ZKMLVerifier.verifyProof(proof, publicSignals) == true
    Guard 3 — consistency:   publicSignals[64] == scoreFieldElement
  getLatestAudit(contractAddress)  → AuditResult
  getAuditHistory(contractAddress) → AuditResult[]
  hasAudit(contractAddress)        → bool
  getAuditCount(contractAddress)   → uint256
  pause() / unpause()              → emergency stop (onlyOwner)

IZKMLVerifier (interface)
  verifyProof(bytes proof, uint256[] publicSignals) → bool
  Concrete implementation is auto-generated by EZKL — see below.

ZKMLVerifier.sol (auto-generated)
  Implements IZKMLVerifier.
  BN254 curve constants embedded from verification_key.vk.
  Must recompile with solc 0.8.17 (Halo2 assembly uses deprecated opcodes).
  Must redeploy when CrossAttentionFusion output_dim changes (currently 128).
```

### `AuditResult` struct

```solidity
struct AuditResult {
    uint256 scoreFieldElement;  // model_output * 8192 (EZKL scale factor)
    bytes32 proofHash;          // keccak256(proof bytes) — not full bytes (gas)
    uint256 timestamp;
    address agent;
    bool    verified;
}
```

Score decoding off-chain: `probability = scoreFieldElement / 8192`.

## ZKMLVerifier — Special Handling

`ZKMLVerifier.sol` is auto-generated by EZKL from `zkml/ezkl/verification_key.vk`. It uses Halo2 assembly with **deprecated opcodes** that require `solc 0.8.17`.

```bash
# 1. Generate verifier (run after zkml/src/ezkl/setup_circuit.py completes)
cp zkml/ezkl/ZKMLVerifier.sol contracts/src/ZKMLVerifier.sol

# 2. Compile with the correct solc version
solc-select install 0.8.17   # first time only
solc-select use 0.8.17
cd contracts && forge build --contracts src/ZKMLVerifier.sol

# 3. Restore default
solc-select use 0.8.20
```

`AuditRegistry` references the verifier via `IZKMLVerifier` (interface, not concrete import) — this makes mixed-solc compilation safe. `AuditRegistry` itself always compiles with 0.8.20.

Must redeploy `ZKMLVerifier` (and update `AuditRegistry`'s verifier address) whenever `CrossAttentionFusion output_dim` changes. Currently **locked at 128**.

## Deployment (Sepolia)

Set environment variables:
```bash
export SEPOLIA_RPC_URL=<your-sepolia-rpc>
export DEPLOYER_PRIVATE_KEY=<your-private-key>
export ETHERSCAN_API_KEY=<your-etherscan-key>
```

Deploy and verify:
```bash
cd contracts
forge script script/Deploy.s.sol \
  --rpc-url $SEPOLIA_RPC_URL \
  --broadcast \
  --verify
```

Deployment order (enforced by `Deploy.s.sol`):
1. Deploy `SentinelToken`
2. Deploy `ZKMLVerifier` (requires `solc 0.8.17` step above)
3. Deploy `AuditRegistry` implementation
4. Deploy ERC-1967 proxy → call `initialize(verifierAddress, tokenAddress)`

After deploy, update these in your environment / `.env`:
- `AUDIT_REGISTRY_ADDRESS` — consumed by `agents/src/mcp/servers/audit_server.py`
- `SENTINEL_TOKEN_ADDRESS` — consumed by staking scripts

## On-chain Interaction

Test ZK proof verification:
```bash
# Generated by zkml/src/ezkl/extract_calldata.py
bash check_verify.sh
```

Submit a full audit:
```bash
# Requires: MIN_STAKE tokens staked + valid proof.json
bash submit_audit.sh
```

Manual queries via `cast`:
```bash
# Check if a contract has been audited
cast call $AUDIT_REGISTRY_ADDRESS \
  "hasAudit(address)(bool)" \
  0x<contract-address> \
  --rpc-url $SEPOLIA_RPC_URL

# Get latest audit result
cast call $AUDIT_REGISTRY_ADDRESS \
  "getLatestAudit(address)((uint256,bytes32,uint256,address,bool))" \
  0x<contract-address> \
  --rpc-url $SEPOLIA_RPC_URL
```

## Critical Constraints

| Constraint | Value | Break condition |
|-----------|-------|----------------|
| solc — AuditRegistry, SentinelToken | **0.8.20** | Compilation failure |
| solc — ZKMLVerifier.sol only | **≤ 0.8.17** | Halo2 assembly opcodes |
| `publicSignals[64]` | Score field element index | `AuditRegistry` Guard 3 check |
| Scale factor | **8192** (2¹³) | Wrong score decoded on-chain |
| Little-endian decode in Python | `int.from_bytes(..., byteorder='little')` | Silent wrong score |
| UUPS initialiser protection | `_disableInitializers()` in constructor | Re-initialisation attack |
| `AuditRegistry` verifier address | Set at `initialize()` | Wrong ZK proof accepted |

## Known Limitations

1. **`AuditResult` stores one `scoreFieldElement`** — designed for a single risk scalar. Track 3 produces 10 per-class scores; a migration design is needed before enabling `submit_audit` in production (see `SENTINEL-SPEC.md §6.5`).
2. **`ZKMLVerifier.sol` not in git** — must be generated from `zkml/ezkl/verification_key.vk` via the EZKL pipeline.
3. **`contracts/out/` not in git** — run `forge build` to produce the ABI needed by `audit_server.py`.
