# Quarantined contracts — broken at the Solidity compile level

**Quarantined:** 2026-06-25
**Reason:** 22 contracts in the original `manual_hand_written_contracts/` corpus
have source code that **any solc version refuses to compile**. These contracts
are not "vulnerable contracts" — they are syntactically broken. ML, Aderyn,
and Slither all fail on the same 22 (verified across solc 0.4.24 through
0.8.31 — see P3 baseline diagnostic `eval/runs/20260626T125014Z_p3_excluded_rule5c/`).

**Why quarantine rather than fix:** Ali decided not to fix the corpus. The
contracts are historical corpus artifacts (some hand-written, some
auto-generated by BCCC injection tooling). The honest measurement is to
exclude them so the eval reports on a corpus that is genuinely evaluable.

**The 4 "slim" bccc contracts that STAY (different failure mode):**
`01_bccc_reentrancy_injected_erc20.sol`, `05_bccc_externalbug_injected_flashloan.sol`,
`06_bccc_timestamp_injected_vesting.sol`, `08_bccc_unusedreturn_injected_batch.sol`.
These compile fine under solc 0.4.x but Slither's version detection picks
solc 0.8.x for them (despite their `pragma ^0.4.24;`). Slither version-detection
bug — fix in Slither, not the corpus.
