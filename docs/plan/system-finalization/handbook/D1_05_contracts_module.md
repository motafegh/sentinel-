**2. AuditRegistry V1** (~0.5 page)
- Struct (verify from `AuditRegistry.sol:28-34`):

**3. AuditRegistry V2** (~1 page)
- Constants (verify from `:38-40`): NUM_CLASSES=10, INPUT_OFFSET=128
- Struct (verify from `:42-56`):
  - uint256[10] classScores — each = round(prob * 8192), little-endian field element
  - bytes32 proofHash, bytes32 modelHash (SHA-256 of teacher checkpoint), uint256 timestamp, address agent, bool verified
- submitAuditV2 (verify from `:140-175`):
  - Guard 1: same stake check
  - Length check: `publicSignals.length >= 138` — BEFORE verifyProof (fail fast with clear error)
  - Guard 2: same ZK proof verification
  - Guard 3: loop `publicSignals[128 + i] == classScores[i]` for i in 0..9
- Event: AuditSubmittedV2(contractAddress, proofHash, agent, classScores[10], modelHash)
- V2 queries: hasAuditV2, getLatestAuditV2, getAuditHistoryV2, getAuditCountV2
- Storage safety: `_auditsV2` is a new mapping — no slot collision with V1 `_audits`
- V1+V2 coexistence: both mappings exist independently, both submit functions work

**4. SentinelToken** (~0.5 page)
- ERC20 with staking (verify from `SentinelToken.sol:10-61`):
  - Constructor: mints 1,000,000 SNTL to deployer
  - MIN_STAKE = 1000 * 10**18 (line 14)
