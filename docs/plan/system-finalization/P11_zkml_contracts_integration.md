### 2.1 The ZK boundary (what the proof proves)

The ZK circuit captures the proxy model computation: `Linear(128→64)→ReLU→Linear(64→32)→ReLU→Linear(32→10)`.

The proof attests: "There exist private weight matrices W_1, W_2, W_3 such that when applied to these specific public inputs (128-dim fusion embedding), the output is exactly these 10 public scores."

Not proved (and not provable at this circuit scale):
- That the 128-dim fusion embedding genuinely came from this specific teacher checkpoint
- That the teacher's GNN+CodeBERT+CrossAttention architecture was correctly applied
- That the source code analyzed matches the on-chain contract

These are covered by the provenance manifest (T11.7) — off-chain auditable claims, not ZK-proven facts.

#### T11.7 — Redesign AuditRegistry for multi-class storage

**What:** Add `AuditResultV2` struct, `submitAuditV2` function, new events, and V2 query functions. Keep V1 (single-score) for backward compatibility.

**`contracts/src/AuditRegistry.sol` — new additions:**

```solidity
// --- V2: multi-class audit result ---

// CLASS_NAMES index mapping (must match graph_schema.py):
//   0: CallToUnknown     5: MishandledException
//   1: DenialOfService    6: Reentrancy
//   2: ExternalBug        7: Timestamp
//   3: GasException       8: TransactionOrderDependence
//   4: IntegerUO          9: UnusedReturn

uint256 public constant NUM_CLASSES = 10;
uint256 public constant INPUT_OFFSET = 128;  // fusion embedding dim

struct AuditResultV2 {
    uint256[10] classScores;  // each = round(prob * 8192), little-endian
    bytes32 proofHash;
    bytes32 modelHash;        // SHA-256 of teacher checkpoint (from provenance manifest)
    uint256 timestamp;
    address agent;
    bool verified;
}

mapping(address => AuditResultV2[]) private _auditsV2;

event AuditSubmittedV2(
    address indexed contractAddress,
    bytes32 proofHash,
    address indexed agent,
    uint256[10] classScores,
    bytes32 modelHash
);

function submitAuditV2(
    address contractAddress,
    uint256[10] calldata classScores,
    bytes calldata proof,
    uint256[] calldata publicSignals,
    bytes32 modelHash
) external whenNotPaused {
    require(
        sentinelToken.stakedBalance(msg.sender) >= sentinelToken.MIN_STAKE(),
        "AuditRegistry: insufficient stake"
    );
    require(
        zkmlVerifier.verifyProof(proof, publicSignals),
        "AuditRegistry: invalid ZK proof"
    );
    // publicSignals layout: [fusion_0..fusion_127, class_score_0..class_score_9]
    // Guard 3: verify all 10 class scores match the proof's public outputs
    for (uint256 i = 0; i < NUM_CLASSES; i++) {
        require(
            publicSignals[INPUT_OFFSET + i] == classScores[i],
            string(abi.encodePacked("AuditRegistry: class score mismatch at index ", i))
        );
    }

    _auditsV2[contractAddress].push(AuditResultV2({
        classScores: classScores,
        proofHash: keccak256(proof),
        modelHash: modelHash,
        timestamp: block.timestamp,
        agent: msg.sender,
        verified: true
    }));

    emit AuditSubmittedV2(contractAddress, keccak256(proof), msg.sender, classScores, modelHash);
}

// V2 queries — mirror V1 interface
function hasAuditV2(address contractAddress) external view returns (bool) {
    return _auditsV2[contractAddress].length > 0;
}
function getLatestAuditV2(address contractAddress) external view returns (AuditResultV2 memory) {
    require(_auditsV2[contractAddress].length > 0, "AuditRegistry: no V2 audits found");
    return _auditsV2[contractAddress][_auditsV2[contractAddress].length - 1];
}
function getAuditHistoryV2(address contractAddress) external view returns (AuditResultV2[] memory) {
    return _auditsV2[contractAddress];
}
function getAuditCountV2(address contractAddress) external view returns (uint256) {
    return _auditsV2[contractAddress].length;
}
```

**Storage slot safety:** `_auditsV2` is a new mapping — does not overlap with V1 `_audits` mapping. No storage collision. UUPS upgradeable.

**Acceptance:** `forge build` succeeds. V1 tests still pass (backward compatible — V1 `submitAudit`, queries, events untouched).

#### T11.12 — Implement provenance manifest signing

**What:** Before submitting `classScores` on-chain, generate a JSON manifest and sign it with the operator's private key (EIP-191). This manifest cryptographically binds the ML model hash to the specific fusion embedding and the resulting class scores — providing the audit trail that the ZK proof alone does not provide (the proof proves the proxy, not the full teacher pipeline).

**Implementation in `audit/_submit.py`:**
```python
import json, hashlib
from eth_account.messages import encode_defunct
from web3.auto import w3

def build_provenance_manifest(
    teacher_model_hash: str,     # from ML API /predict response
    proxy_checkpoint_hash: str,  # SHA-256 of proxy_best.pt
    fusion_embedding: list[float],
    class_scores: list[float],
    operator_address: str,
) -> dict:
    fusion_hash = hashlib.sha256(
        json.dumps(fusion_embedding, sort_keys=True).encode()
    ).hexdigest()

    manifest = {
        "teacher_model_hash": teacher_model_hash,
        "proxy_checkpoint_hash": proxy_checkpoint_hash,
        "fusion_embedding_hash": fusion_hash,
        "class_scores": [round(s, 6) for s in class_scores],
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "operator_address": operator_address,
    }
    message = encode_defunct(text=json.dumps(manifest, sort_keys=True))
    signed = w3.eth.account.sign_message(message, private_key=OPERATOR_KEY)
    manifest["signature"] = signed.signature.hex()
    return manifest
```

**Verification (off-chain):** Anyone can verify: `json.loads(message) → EIP-191 verify(signature, message) → True` and confirm `teacher_model_hash` matches the known checkpoint, `fusion_embedding_hash` matches the stored fusion vector, etc.

**Acceptance:** Manifest is valid EIP-191 signed JSON. Signature verifies against operator_address. Included in `final_report["on_chain"]["provenance"]`.

## 8. What this plan deliberately defers

- **Run 13 / GasException removal:** Deferred per Ali. ZKML circuit is locked to 10 classes (current reality). If Run 13 later drops GasException → CIRCUIT_VERSION v3.0 → full EZKL regen → new ZKMLVerifier deployment. The architecture supports this cleanly.
- **Full teacher in ZK circuit:** Infeasible at 125M params vs ~10K circuit limit. Provenance manifest provides auditable binding instead of cryptographic binding.
- **Sepolia → Mainnet migration:** Same contracts, different deploy address. Not in scope.
- **Agent staking automation:** `MIN_STAKE` check is in the contract Guard 1. Funding the operator key and staking is an operator concern, not automated in this plan.
- **`modelHash` in V1 `AuditResult`:** V1 struct is frozen for backward compatibility. V2 `AuditResultV2` includes `modelHash`. V1 consumers see single-score; V2 consumers see multi-class + provenance.
- **Gas optimization of Guard 3 loop:** 10 `require` statements. Gas impact: ~5k per require = ~50k additional. Acceptable for an audit oracle (submissions are infrequent). Premature optimization.
- **Proof batch aggregation:** One proof per audit. If later needed for throughput, batch multiple contracts into one proof. Circuit would need redesign — defer.
