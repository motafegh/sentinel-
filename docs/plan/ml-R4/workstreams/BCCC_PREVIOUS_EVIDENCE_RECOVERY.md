# BCCC Previous-Evidence Recovery Workstream

## Purpose

Recover BCCC review evidence, retrospective cleanup, and model-run lineage without assuming they were the training labels.

## Search targets

- original label/split files;
- source acquisition;
- review batches;
- embedded source snippets;
- verification reports;
- class-size tables;
- KEEP/DROP/REVIEW decisions;
- tests with historical expected counts;
- benchmark reports;
- temporary output references;
- checkpoints trained on BCCC;
- export hashes and training configs.

## Key distinctions

- original training labels versus later reviewed/cleaned outputs;
- raw evidence versus conclusion-only reports;
- source snippet retained versus source unavailable;
- regression expectation versus reproducible audit;
- new reproduction versus historical artifact.

## New review rule

Missing BCCC artifacts create a proposed gap. They do not automatically authorize a broad replacement audit.
