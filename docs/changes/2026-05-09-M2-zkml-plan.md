## 2. The Decision: Option A vs Option B (must precede any other M2 work)

### Option A — Run the pipeline against v3 (or wait for v4)

Pros:
- Closes the only "source-complete-but-not-run" half-loop in the project
- Unblocks M5 deploy (`ZKMLVerifier.sol` is auto-generated; M5 currently
  references `IZKMLVerifier` only)
- Shipping value of SENTINEL is the on-chain verifiable audit; without
  M2 the chain branch is decorative

Cons / costs:
- EZKL setup is environment-heavy (Rust toolchain, GPU memory budget for
  setup ~20 GB, Halo2 SRS download)
- Setup step alone is single-digit hours on first run
- If v4 lands soon, repeating the pipeline doubles effort (architecture is
  frozen, but proxy weights change, so steps 1–5 must rerun if the proxy
  retrains; v2.0 keys remain valid only if the proxy weights distil into
  the same circuit)

### Option B — Descope to S10

Pros:
- No environment friction in the current sprint
- Lets M5/M6 ship with `MockZKMLVerifier` and a stubbed verify path

Cons:
- Project's headline differentiator (ZK-verified audits) becomes future-tense
- Submitting on-chain in production without verification is unsafe; M5
  must keep `submitAudit` paused or behind a feature flag

### Recommendation

Pick **Option A**, but **schedule it after v4 retrain lands**. Distill
once against the final teacher to avoid wasted setup work. Write the ADR
either way (see §3).

## 4. Plan A.1 — Distillation against v3 (only if Option A chosen, gated on v4)

Sequenced steps with file references:

### 4.1 Train proxy

```bash
poetry run python zkml/src/distillation/train_proxy.py \
    --teacher-checkpoint ml/checkpoints/multilabel-v3-fresh-60ep_best.pt \
    --output zkml/checkpoints/proxy_v2_v3teacher.pt \
    --target-agreement 0.95
```

Acceptance: per-class agreement ≥ 95 % vs teacher on val split.
If below: fall back to a wider hidden layer is **NOT** allowed — that
breaks `CIRCUIT_VERSION` lock (ADR-007). Instead increase distillation
epochs / vary loss weighting.

### 4.2 Export ONNX

```bash
poetry run python zkml/src/distillation/export_onnx.py \
    --proxy zkml/checkpoints/proxy_v2_v3teacher.pt \
    --output zkml/onnx/proxy_v2_v3teacher.onnx
```

Acceptance: ONNX checker passes, dynamic batch axis present, no opset
warnings, `dummy_input.shape == (1, 128)`.

### 4.3 EZKL setup (one-time, expensive)

```bash
poetry run python zkml/src/ezkl/setup_circuit.py \
    --onnx zkml/onnx/proxy_v2_v3teacher.onnx \
    --calibration zkml/calibration/calibration_v3.json
```

Outputs: `zkml/ezkl/proving_key.pk` (gitignored), `verification_key.vk`,
`ZKMLVerifier.sol`.

### 4.4 Generate calibration data

```bash
poetry run python zkml/src/distillation/generate_calibration.py \
    --teacher-checkpoint ml/checkpoints/multilabel-v3-fresh-60ep_best.pt \
    --output zkml/calibration/calibration_v3.json \
    --num-samples 100
```

Run this **before** §4.3 (setup needs the calibration file).

### 4.5 Per-audit proof smoke test

```bash
poetry run python zkml/src/ezkl/run_proof.py
```

Acceptance: `proof.json` produced, EZKL `verify()` returns True, score
field-element decodes (little-endian) to within 1 LSB of teacher score
× 8192.

### 4.6 Solidity verifier handoff

- Copy generated `ZKMLVerifier.sol` to `contracts/src/`
- See M5 plan §3 for solc-select 0.8.17 build path
