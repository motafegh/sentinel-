## Fixes Applied

### `ml/src/models/gnn_encoder.py`

#### Fix A1 — Residual dropout order (correctness)
**Finding:** `x = self.dropout(x2 + x)` applies dropout to the residual sum, meaning the skip
connection itself can be zeroed. The ResNet principle requires the skip to be an identity path.

**Fix:** `x = x + self.dropout(x2)` — dropout applied only to the transformation branch.

```python
# Before
x = self.dropout(x2 + x)
# After
x = x + self.dropout(x2)
```

Applies to Phase 1 (line ~436 at time of fix).

#### Fix A2 — `_JKAttention.last_weights` promoted to `register_buffer`
**Finding:** `self.last_weights` was a plain Python attribute, not a PyTorch buffer. It would
not survive `.to(device)`, would not be saved/loaded in `state_dict()`, and would break under
DDP. It was also reset on every forward call using `=` assignment, which does not update a
tensor in-place and silently falls back to a fresh attribute each call.

**Fix:**
```python
def __init__(self, channels: int, num_phases: int = 3) -> None:
    super().__init__()
    self.attn = nn.Linear(channels, 1, bias=False)
    self.register_buffer("last_weights", torch.zeros(num_phases))
```
```python
# In forward — use copy_ for in-place update of registered buffer
self.last_weights.copy_(weights.squeeze(-1).mean(0).detach())
```

#### Fix A3 — Degraded-mode warning when edge_attr is None
**Finding:** When `use_edge_attr=True` (the v5.2 default) but a graph has no edge attributes,
Phase 2 and Phase 3 are silently skipped. A caller has no way to detect this.

**Fix:** Added a `logger.warning(...)` log when entering degraded mode:
```python
if self.use_edge_attr:
    logger.warning(
        "GNNEncoder: use_edge_attr=True but edge_attr is None. "
        "Phase 2 (CONTROL_FLOW) and Phase 3 (REVERSE_CONTAINS) are disabled. "
        "Only Phase 1 embeddings will be returned."
    )
```

### `ml/src/models/sentinel_model.py`

#### Fix B1 — Empty batch guard in forward
**Finding:** If `batch` tensor is empty (numel == 0), calling `global_max_pool(node_embs, batch)`
followed by `.max()` on the result raises `RuntimeError` with an unhelpful message.

**Fix:** Guard added at the start of the GNN pooling block:
```python
if batch.numel() == 0:
    return torch.zeros(B, self.eye_dim, device=node_embs.device)
```

#### Fix B2 — `size=num_graphs` on pool calls
**Finding:** `global_max_pool` and `global_mean_pool` without `size=` return a tensor with
as many rows as there are unique values in `batch`. If a graph in a batch has no FUNCTION
nodes (only CFG nodes), it would be omitted, producing a result with fewer rows than `B`.
This causes a shape mismatch in the three-eye concat.

**Fix:**
```python
gnn_eye = global_max_pool(func_embs, func_batch, size=num_graphs)  # [B, hidden]
```
Applied to both pool calls.

#### Fix B3 — Remove per-forward `logger.debug()`
**Finding:** A `logger.debug(...)` call was emitted on every forward pass, which adds ~1 μs
overhead per step and floods debug logs making them useless for actual debugging.

**Fix:** Call removed.

#### Fix B4 — Aux head squeeze for binary classification (num_classes == 1)
**Finding:** When `num_classes == 1`, the main logits are squeezed to `[B]` via
`logits.squeeze(1)`. The auxiliary heads (`aux_gnn`, `aux_tf`, `aux_fused`) were not squeezed,
leaving them at `[B, 1]` — shape mismatch against labels `[B]` in the aux loss.

**Fix:**
```python
if self.num_classes == 1:
    aux_gnn   = aux_gnn.squeeze(1)
    aux_tf    = aux_tf.squeeze(1)
    aux_fused = aux_fused.squeeze(1)
```

### `ml/src/training/trainer.py`

#### Fix C1 — Module-level environment/backend mutation moved inside `train()`
**Finding:** `os.environ["TRANSFORMERS_OFFLINE"] = "1"`, `torch.backends.cuda.matmul.allow_tf32`,
`torch.backends.cudnn.allow_tf32`, and `torch.backends.cudnn.benchmark` were set at module
import time. Any process that imports trainer (tests, tune_threshold, API) silently inherits
these mutations, even if they never intend to train.

**Fix:** All four moved inside `train()`, guarded by device type:
```python
os.environ["TRANSFORMERS_OFFLINE"] = "1"
os.environ["HF_HUB_OFFLINE"]       = "1"
if device == "cuda":
    torch.backends.cuda.matmul.allow_tf32 = True
    torch.backends.cudnn.allow_tf32       = True
    torch.backends.cudnn.benchmark        = True
```

#### Fix C2 — `autocast` device guard (CPU crash prevention)
**Finding:** `torch.amp.autocast("cuda", ...)` is hardcoded in both `train_one_epoch()` and
`evaluate()`. Running on CPU (smoke tests, CI) raises `RuntimeError: CUDA is not available`.

**Fix:** `autocast(device, ...)` — `device` is already a string variable in scope:
```python
with torch.amp.autocast(device, enabled=use_amp):
```

#### Fix C3 — Scheduler step guarded by GradScaler scale check
**Finding:** When `GradScaler` detects an inf/NaN gradient, `scaler.step(optimizer)` is
internally skipped (no parameter update). However, `scheduler.step()` still runs, consuming
a learning-rate step without any training progress. This makes the LR schedule drift ahead of
the actual number of optimizer steps, ultimately causing premature LR decay.

**Fix:**
```python
_scale_before = scaler.get_scale()
scaler.step(optimizer)
scaler.update()
if scaler.get_scale() == _scale_before:   # scale unchanged → step was not skipped
    scheduler.step()
```

#### Fix C4 — `gnn_eye_proj` missing from GNN param group
**Finding:** The GNN LR multiplier group was built by filtering `pname.startswith("gnn.")`.
`gnn_eye_proj` — the projection head from the GNN pooling output to the classifier — starts
with `gnn_eye_proj.` not `gnn.`, so it was silently placed in the default group at 1.0× LR,
receiving weaker gradient signal than the GNN encoder it feeds.

**Fix:**
```python
if _pname.startswith("gnn.") or _pname.startswith("gnn_eye_proj."):
    _group = "gnn"
```

#### Fix C5 — Scheduler state only restored when optimizer was also restored
**Finding:** On resume with `--no-resume-model-only`, the flag `not _skip_optimizer` controlled
whether the scheduler was restored. But `_skip_optimizer` was a logic flag that could be True
even after a successful optimizer load (e.g., param-group mismatch fallback path). Restoring
scheduler state after a partial optimizer load misaligns the LR schedule.

**Fix:** Introduced explicit `_optimizer_restored = False` flag; set to `True` only on clean
optimizer restore. Scheduler restore now gated on `_optimizer_restored`:
```python
if "scheduler" in ckpt and _optimizer_restored:
    scheduler.load_state_dict(ckpt["scheduler"])
```

#### Fix C6 — GNN gradient share formula: RMS norm not sum-of-norms
**Finding:** The share formula was `gnn_norm / (gnn_norm + tf_norm + fused_norm)`. The
denominator is the sum of three independent norms. It over-penalises GNN when TF and fused
norms are large, reporting 12–18% share even when GNN gradients are healthy (30%+). The
correct vector-magnitude formula is RMS: `sqrt(gnn² + tf² + fused²)`.

**Fix:**
```python
_total_norm = (gnn_norm**2 + tf_norm**2 + fused_norm**2) ** 0.5
_gnn_share  = gnn_norm / (_total_norm + 1e-8)
```

#### Fix C7 — JSON crash protection on `.state.json` read
**Finding:** `_resume_state_path.read_text()` followed by `json.loads()` has no error handling.
A partially-written sidecar (process killed mid-write, disk full) would raise `JSONDecodeError`
and crash the entire resume, destroying a valid checkpoint.

**Fix:**
```python
try:
    _saved_state = json.loads(_resume_state_path.read_text())
except json.JSONDecodeError as _e:
    logger.warning(f"Corrupt .state.json ({_e}) — patience_counter from checkpoint.")
    _saved_state = {}
```

#### Fix C8 — Atomic checkpoint save
**Finding:** `torch.save({...}, checkpoint_path)` writes to the final path directly. If the
process is killed mid-write, the checkpoint file is left truncated/corrupt with no way to
recover. The previous good checkpoint is also gone (overwritten by the partial write).

**Fix:** Write to a `.tmp` file then atomically rename:
```python
_tmp_path = checkpoint_path.with_suffix(".tmp")
torch.save({...}, _tmp_path)
_tmp_path.replace(checkpoint_path)   # POSIX atomic rename
```

#### Fix I1 — `mp.set_start_method` at module level
**Finding:** `mp.set_start_method('spawn', force=True)` at module level runs on every import of
`train.py` — including in tests, the REPL, and any tool that imports from scripts/. On macOS
and Linux this is harmless but can raise `RuntimeError` in some multiprocessing contexts.

**Fix:** Moved inside `if __name__ == "__main__":`, before `main()`:
```python
if __name__ == "__main__":
    mp.set_start_method('spawn', force=True)
    main()
```

#### Fix C1 (H9) — `gnn_encoder.py`: OOB edge_attr clamp before `nn.Embedding`
**Finding:** Corrupted `.pt` files with `edge_attr` values outside `[0, NUM_EDGE_TYPES-1]`
cause `nn.Embedding` to throw an unhelpful index-out-of-range CUDA error with no indication
of which contract caused it. Training crashes unrecoverably on a single bad file.

**Fix:** Added bounds check before the embedding lookup. Any OOB values are clamped to
`[0, NUM_EDGE_TYPES-1]` with a logged warning (count, observed min/max, advice to
re-extract). The forward pass continues rather than crashing the training run.

```python
if edge_attr.numel() > 0:
    _oob_mask = (edge_attr < 0) | (edge_attr >= NUM_EDGE_TYPES)
    if _oob_mask.any():
        logger.warning(f"GNNEncoder: {_oob_mask.sum().item()} OOB edge_attr value(s) ...")
        edge_attr = edge_attr.clamp(0, NUM_EDGE_TYPES - 1)
e = self.edge_embedding(edge_attr)
```

**Note:** The clamp also fixes the downstream edge masks (`struct_mask`, `cfg_mask`,
`contains_mask`) since they operate on the same `edge_attr` tensor.

#### Fix D1 (H14) — `dual_path_dataset.py`: random 10-hash integrity sample
**Finding:** The RAM cache spot-check verified only `paired_hashes[0]` — the first hash in
sorted order. This hash tends to be stable across builds (same MD5 for the same source file),
so a partial rebuild where most hashes changed would pass the check undetected.

**Fix:** Replaced the single spot-check with a random sample of 10 hashes using
`random.sample(self.paired_hashes, min(10, len(self.paired_hashes)))`. Each sampled hash is
checked for presence in the cache dict and for structural validity (`graph.x`, `tokens["input_ids"]`).

```python
_sample_size   = min(10, len(self.paired_hashes))
_sample_hashes = random.sample(self.paired_hashes, _sample_size)
for _spot in _sample_hashes:
    if _spot not in self.cached_data:
        raise RuntimeError(f"RAM cache is stale — hash {_spot!r} not found ...")
    _g, _t = self.cached_data[_spot]
    ...  # structural validation
```

#### Fix D2 (H15) — `dual_path_dataset.py` + `create_cache.py`: schema version in cache
**Finding:** The cache pickle stored only `md5 → (graph, tokens)` pairs with no metadata.
After incrementing `FEATURE_SCHEMA_VERSION` (e.g., v2→v3 on 2026-05-12), the existing cache
would be silently loaded and fed to the model, causing accuracy regression with no error signal.

**Fix (two-file):**

`create_cache.py` now embeds `FEATURE_SCHEMA_VERSION` in the cache dict before pickling:
```python
cached["__schema_version__"] = FEATURE_SCHEMA_VERSION  # e.g., "v3"
```

`dual_path_dataset.py` validates it on load:
```python
_cached_schema = self.cached_data.get("__schema_version__")
if _cached_schema is not None and _cached_schema != _FEATURE_SCHEMA_VERSION:
    raise RuntimeError(f"RAM cache schema mismatch: cache={_cached_schema!r} != current={_FEATURE_SCHEMA_VERSION!r} ...")
elif _cached_schema is None:
    logger.warning("RAM cache has no '__schema_version__' key — may be stale. Rebuild with create_cache.py.")
```

Backward-compatible: old caches without the key produce a warning, not a crash.

#### Fix E1 (C4) — `inference/preprocess.py`: `_tokenize_sliding_window` CLS/SEP fix
**Finding:** The sliding-window tokenizer encoded the full sequence with
`add_special_tokens=True` (`[CLS] c1 c2 … cN [SEP]`), then sliced it with a fixed stride.
Non-first windows (e.g., `full_ids[256:768]`) produced bare token sequences — no `[CLS]` at
position 0, no `[SEP]` at the end. CodeBERT was pre-trained to always see `[CLS]…[SEP]`;
without `[CLS]`, the transformer's contract-level representation (used by the "tf eye") comes
from a random mid-contract token rather than the learned classification embedding.

**Fix:** Encode with `add_special_tokens=False` to get pure content tokens, then wrap each
window individually with `[CLS]` + `[SEP]`. Content window capacity is `MAX_TOKEN_LENGTH - 2`
(510 tokens) to leave room for the two special tokens.

```python
# Before (wrong): sliced the full special-token sequence
full_ids = self.tokenizer.encode(source_code, add_special_tokens=True)
chunk    = full_ids[start:end]  # no [CLS]/[SEP] for windows after the first

# After (correct): wrap every window
full_content_ids = self.tokenizer.encode(source_code, add_special_tokens=False)
_CONTENT_CAP     = self.MAX_TOKEN_LENGTH - 2  # 510
content_chunk    = full_content_ids[start : start + _CONTENT_CAP]
chunk            = [cls_id] + content_chunk + [sep_id]  # every window is properly framed
```

Fast path (`total_content <= 510`) unchanged — delegates to `_tokenize()` which uses the
standard tokenizer call and is already correct.

#### Fix F1 (C7) — `tune_threshold.py`: `_ensure_list()` for `lora_target_modules`
**Finding:** `ckpt_config.get("lora_target_modules", ["query", "value"])` could return a
non-list from old checkpoints (string, tuple, or absent). Passing a raw string to PEFT causes
it to silently treat the entire string as a single module name — e.g., `"query,value"` would
match nothing, disabling LoRA entirely with no error.

**Fix:** Added `_ensure_list()` helper that normalises any value to a Python list:

```python
def _ensure_list(value: Any, default: list) -> list:
    if value is None:         return default
    if isinstance(value, list):  return value
    if isinstance(value, (tuple, set)): return list(value)
    if isinstance(value, str):  return [value]
    return list(value)

# Usage:
lora_target_modules=_ensure_list(
    ckpt_config.get("lora_target_modules"),
    default=["query", "value"],
),
```

## Confirmed Invalid Findings

| Finding | Why Invalid |
|---|---|
| `squeeze(-1)` cache mutation in JK | `squeeze(-1)` on a 1-D tensor is a no-op; no mutation occurs |
| edge_attr `squeeze` aliasing | edge_attr is read-only in the dataset; DataLoader creates new tensors per batch |
| Checkpoint dict import cycle | Checkpoint is a plain dict, not a module import; no cycle possible |
| Warmup graph `edge_attr` shape | Warmup creates minimal graph explicitly; shape is `[E]` int64 as required |
| `BCEWithLogitsLoss` sigmoid double-application | Model has NO sigmoid; loss applies it internally — correct |
| Focal loss `gamma=0` degeneration | Focal loss is an optional path; v5.2 uses BCE. Finding applies to disabled code |
