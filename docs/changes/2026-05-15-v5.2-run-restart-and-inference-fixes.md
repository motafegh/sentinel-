## Fix: `gnn_use_jk` missing in predictor.py

**File:** `ml/src/inference/predictor.py`

**Problem:** `SentinelModel()` was built without `gnn_use_jk`, so loading a v5.2
checkpoint (which has JK attention weights in the state dict) would fail with a
`load_state_dict()` unexpected-keys error.

**Fix:** Added `gnn_use_jk=saved_cfg.get("gnn_use_jk", _is_v5)` to the
`SentinelModel()` call. Defaults to `True` for `three_eye_v5` architecture
(safe fallback if checkpoint predates the `gnn_use_jk` config key).

---

## Fix: `gnn_use_jk` and v5 defaults missing in tune_threshold.py

**File:** `ml/scripts/tune_threshold.py`

**Problem:** `load_model_from_checkpoint()` was missing `gnn_use_jk` (same crash
as predictor). Also used v4-era defaults (`gnn_hidden_dim=64`, `gnn_edge_emb_dim=16`,
`lora_r=8`, `lora_alpha=16`) regardless of architecture, so v5.2 models would be
built with wrong dimensions before the state dict mismatch was even reached.

**Fix:**
- Added `_is_v5 = (architecture == "three_eye_v5")` flag
- Added `gnn_num_layers=ckpt_config.get("gnn_layers", 4)`
- Updated all dimension defaults to be v5-aware:
  - `gnn_hidden_dim`: 128 if v5, 64 otherwise
  - `gnn_edge_emb_dim`: 32 if v5, 16 otherwise
  - `lora_r`: 16 if v5, 8 otherwise
  - `lora_alpha`: 32 if v5, 16 otherwise
- Added `gnn_use_jk=ckpt_config.get("gnn_use_jk", _is_v5)`
