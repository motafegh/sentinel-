## Phase 0 — Immediate Code Fixes

### P0-1: test_gnn_encoder.py — Complete Rewrite

**File:** `ml/tests/test_gnn_encoder.py`

**Problem:** The test helper `_make_graph()` created node features with dim=8 (v1 schema) but
`GNNEncoder` defaults to `in_channels=NODE_FEATURE_DIM=12`. The dim mismatch would crash at the
first GATConv layer, meaning no test here could catch a real forward-pass regression. Two tests
also documented the wrong contract: `test_graceful_degradation_none_edge_attr` expected no raise,
but `gnn_encoder.py` raises `ValueError` for `edge_attr=None + use_edge_attr=True`; and
`test_zero_fallback_differs_from_real_edge_attr` passed `edge_attr=None` which also raised.

**Fix:** Full rewrite with change history comments. Key changes:
- `_make_graph()` now uses `torch.randn(n_nodes, NODE_FEATURE_DIM)` (imported constant)
- `test_edge_attr_none_raises_when_expected` — asserts `ValueError` is raised (correct contract)
- `test_edge_attr_influences_output` — tests with two different valid edge_attr tensors
- `test_no_edge_graph` — uses `NODE_FEATURE_DIM`

**New tests added (Phase 1 gates, written before implementation):**
- `test_jk_gradient_flow` — **NON-NEGOTIABLE GATE**: every JK and LayerNorm param must have
  non-zero gradient after backward. Catches the detach bug that would make JK inert during training.
- `test_reverse_contains_separate_embedding` — REVERSE_CONTAINS (type 7) embedding must differ
  from CONTAINS (type 5) after random init.
- `test_jk_output_shape_unchanged` — JK attention mode must output `[N, hidden_dim]`
- `test_jk_disabled_output_shape` — `use_jk=False` must output `[N, hidden_dim]`

**Result:** 11/11 tests pass.

## Phase 1 — Architecture Changes

### P1-A3: REVERSE_CONTAINS — graph_schema.py + gnn_encoder.py

**Files:** `ml/src/preprocessing/graph_schema.py`, `ml/src/models/gnn_encoder.py`

**Problem (L2):** Phase 3 reused the forward CONTAINS (type 5) embedding for the reversed
CFG_NODE → FUNCTION direction. The GNN could not learn directional asymmetry from edge attributes
alone. Confirmed in code at `gnn_encoder.py` (old line 280).

**graph_schema.py changes:**
- `NUM_EDGE_TYPES` bumped **7 → 8**
- `EDGE_TYPES["REVERSE_CONTAINS"] = 7` added
- Schema history updated — this is a **v3 change** (schema HISTORY comment updated)
- `assert len(EDGE_TYPES) == NUM_EDGE_TYPES` still enforced

**Key constraint — no re-extraction needed:** Type 7 is **runtime-only**. It is generated inside
`GNNEncoder.forward()` by flipping CONTAINS(5) edges and assigning type-7 IDs on the fly.
Graph `.pt` files on disk never contain id=7. The only impact on disk is one extra row in the
`nn.Embedding(8, 32)` table (was `Embedding(7, 32)`).

**gnn_encoder.py Phase 3 change:**
```python
# Before (L2 bug): reused type-5 embedding for reversed edges
rev_contains_ea = e[contains_mask]

# After (fixed): generates type-7 IDs and embeds them distinctly
n_rev = rev_contains_ei.shape[1]
rev_type_ids    = torch.full((n_rev,), _REVERSE_CONTAINS, dtype=torch.long, device=...)
rev_contains_ea = self.edge_embedding(rev_type_ids)
```

### P1-A1+A2: JK Connections + Per-Phase LayerNorm

**File:** `ml/src/models/gnn_encoder.py`

**Problem (L3):** By epoch 8 in v5.1-fix28 the GNN gradient share dropped to ~10% — the only
output fed to the classifier was Phase 3 (over-smoothed, losing Phase 1 structural signal).
There was no mechanism for the classifier to directly access the Phase 1 or Phase 2 embeddings.

**New: `_JKAttention` module** (inner class, not PyG's JumpingKnowledge):
```python
class _JKAttention(nn.Module):
    def __init__(self, channels: int):
        self.attn = nn.Linear(channels, 1, bias=False)

    def forward(self, xs: list[torch.Tensor]) -> torch.Tensor:
        stacked = torch.stack(xs, dim=1)        # [N, K, channels]
        scores  = self.attn(stacked)             # [N, K, 1]
        weights = torch.softmax(scores, dim=1)   # [N, K, 1]
        self.last_weights = weights.squeeze(-1).mean(0).detach()  # [K] cached for monitoring
        return (weights * stacked).sum(dim=1)    # [N, channels]
```

Output dimension is exactly `hidden_dim` — no downstream shape change.

**CRITICAL — live intermediates:**
JK aggregation consumes `_live = []`, collected **without** `.detach()`. The existing
`_intermediates` dict (for `return_intermediates=True`) keeps `.detach().clone()` for backward
compatibility. Mixing these would have given JK zero gradients for the entire training run (the
original detach bug confirmed by `test_jk_gradient_flow`).

**Per-phase LayerNorm** (`self.phase_norm = nn.ModuleList([LayerNorm(128) for _ in range(3)])`):
Applied after each phase's residual connection, before collecting for JK. Without LayerNorm,
Phase 1 (two conv layers + residual) produces higher norms than Phases 2+3 (one layer each),
causing JK softmax to ignore later phases regardless of learned weights.

**New constructor params:** `use_jk: bool = True`, `jk_mode: str = 'attention'`

## Phase 2 — Training Pipeline Changes

### P2-B1: Separate LR Groups

**File:** `ml/src/training/trainer.py`

**Problem:** Single AdamW group for all trainable params. GNN receives same LR as LoRA despite
needing a stronger update signal to prevent gradient collapse.

**Change:** Three param groups, built by iterating `model.named_parameters()`:
| Group | Filter | LR multiplier | Default effective LR |
|---|---|---|---|
| GNN | `name.startswith("gnn.")` | `gnn_lr_multiplier = 2.5` | `5e-4` |
| LoRA | `"lora_" in name` | `lora_lr_multiplier = 0.5` | `1e-4` |
| Other | everything else | 1.0 | `2e-4` |

- `gnn_lr_multiplier` and `lora_lr_multiplier` added to `TrainConfig`
- `OneCycleLR` receives `max_lr=[gnn_lr, lora_lr, other_lr]` (per-group list)
- On resume: guard checks `len(ckpt["optimizer"]["param_groups"]) == len(optimizer.param_groups)`;
  if mismatch (1 vs 3 from pre-v5.2 checkpoint), skips optimizer restore with WARNING

### P2-B3: NaN Loss Counter

**File:** `ml/src/training/trainer.py`

`GradScaler` silently skips optimizer steps when loss is NaN/Inf. Previously invisible.

- `nan_loss_count` tracked per epoch in `train_one_epoch`
- Logged to MLflow as `nan_batch_count` each epoch
- `train_one_epoch` return signature changed to `(avg_loss, nan_count)` tuple
- Warning logged if any NaN batches occurred that epoch

### P2-B4: Orphan-Token Log Downgrade

**File:** `ml/src/datasets/dual_path_dataset.py`

The `{N} token files have no matching graph file` log was `WARNING` — it fired every run with
the same ~24K number, polluting logs with a non-actionable message.

Changed to `DEBUG`. (The 24K files have since been physically moved — see Data section below.)

### P2-C1: JK Attention Weight Logging

**Files:** `ml/src/models/gnn_encoder.py`, `ml/src/training/trainer.py`

`_JKAttention.forward()` caches `self.last_weights` — a detached `[K]` tensor of mean per-phase
attention weights from the last training batch.

After each epoch, `trainer.py` reads this and logs three MLflow metrics:
- `jk_phase1_weight`, `jk_phase2_weight`, `jk_phase3_weight`

Also logs human-readable line: `JK attention weights — Phase1=X Phase2=Y Phase3=Z`

**v5.2 smoke gate:** all three weights must be > 5% after smoke run.

### P2-C2: GNN Collapse Early Warning (Improved)

**File:** `ml/src/training/trainer.py`

Previous check: `gnn_norm / tf_norm < 0.05` — a ratio that gives false positives when tf_norm
is very small, and doesn't account for the fused eye at all.

New check: **total share** = `gnn_norm / (gnn_norm + tf_norm + fused_norm)`, threshold 10%.
Tracks a **streak counter** (`_gnn_collapse_streak`): WARNING only fires after 3 consecutive
log intervals below threshold, matching the epoch-8 collapse pattern from v5.1-fix28.
Single intervals below 10% log at INFO `[streak N/3]`.

### P2-C3: Phase Dominance Alert

**File:** `ml/src/training/trainer.py`

After per-epoch JK weight logging, if any phase has `weight > 0.80` a WARNING is logged:
```
⚠ JK phase dominance: Phase N has XX.X% attention weight.
Other phases are underutilised — consider checking LayerNorm or LR.
```
