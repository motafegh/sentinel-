## 4. Plan C — Source Gaps to Close Before Deploy

### 4.1 Missing `submitAudit` re-entrancy guard

`AuditRegistry.sol` does not (per current source survey) include
`ReentrancyGuardUpgradeable`. Even though `submitAudit` is mostly storage
writes, the `IZKMLVerifier.verifyProof` call is to an external contract
that could be malicious in a future swap. Add:

```solidity
import "@openzeppelin/contracts-upgradeable/utils/ReentrancyGuardUpgradeable.sol";
contract AuditRegistry is ..., ReentrancyGuardUpgradeable {
    function initialize(...) ... { ... __ReentrancyGuard_init(); }
    function submitAudit(...) ... whenNotPaused nonReentrant { ... }
}
```

Test addition: malicious mock verifier that re-enters `submitAudit`.

### 4.2 Slashing → token burn or treasury

Verify `SentinelToken.slash(agent, amount)` semantics. Spec says
"penalise dishonest agent"; source must explicitly burn (or move to
treasury). If currently burns: document; if transfers to owner: convert
to burn or to a configurable treasury address (add a setter `onlyOwner`).

### 4.3 Event indexing

Confirm `AuditSubmitted` indexes both `contractAddress` and `agent`
(spec confirms it does — verify against current source). Add a test that
filters `AuditSubmitted` by both indexed fields and asserts logs return.

### 4.4 UUPS authorisation

Check `_authorizeUpgrade(address newImplementation)` is `onlyOwner`. Add
test: non-owner upgrade attempt reverts; owner upgrade succeeds; emits
`ImplementationUpgraded`.

## 5. Plan D — Deployment (Sepolia)

Only after §2–§4 are green AND M2 Option A is decided.

### 5.1 Pre-deploy checklist

- [ ] All forge tests pass on `lib`-installed deps
- [ ] Slither run clean (or all findings classified as Info / Acknowledged)
- [ ] `MIN_STAKE` value confirmed in source vs spec (1000 SNTL)
- [ ] Sepolia RPC + deployer key in env (`SEPOLIA_RPC_URL`,
      `DEPLOYER_PRIVATE_KEY`); never commit
- [ ] Etherscan key in env for verify (`ETHERSCAN_API_KEY`)
- [ ] M2 produced `ZKMLVerifier.sol` (Option A path) — copied to `contracts/src/`

### 5.2 Deploy command

```bash
cd contracts
forge script script/Deploy.s.sol \
    --rpc-url $SEPOLIA_RPC_URL \
    --broadcast \
    --verify
```

Post-deploy:
- Record `SentinelToken` and `AuditRegistry proxy` addresses in
  `contracts/README.md` (section "Deployed Addresses")
- Record `ZKMLVerifier` address (if Option A) and the
  `proxyAdmin/initial owner` address
- Update `agents/.env` with the AuditRegistry proxy address

### 5.3 Post-deploy sanity

```bash
cast call $REGISTRY_ADDR "zkmlVerifier()(address)"
cast call $TOKEN_ADDR "MIN_STAKE()(uint256)"
cast call $REGISTRY_ADDR "owner()(address)"
```

Each must return the expected value.
