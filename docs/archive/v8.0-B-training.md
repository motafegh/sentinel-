## Why This Run Exists

Three architecture ablations (v7.0, v8-AB, PLAN-3A) all converged to the same
~0.287 tuned F1-macro ceiling despite significant architecture differences. The
conclusion: the bottleneck is data quality, not architecture.

This run is the **first post-data-quality-fix training run**. Same architecture
as PLAN-3A (the best-found configuration), with two changes:

1. **Cleaner labels** — label_cleaner.py tightened across all classes:
   - Stricter Reentrancy check: requires ext_call AND WRITES edge (was ext_call only)
   - Stricter Timestamp check: requires block globals AND (ext_call OR payable)
   - New ExternalBug check: requires ext_call > 0 (14.8% had none)
   - **Total removed: 4,304 labels** across 6 classes
2. **DoS gradient re-enabled** — `dos_loss_weight=0.5` instead of 0.0. DoS now
   has ~243 training positives; the original 0.0 was set when DoS had 3 samples.

## Hypotheses

### H1 — Reentrancy will improve (+0.01–0.03 vs PLAN-3A 0.291)
**Basis:** 611 mislabeled contracts removed (no-ext-call false positives).
The model now trains on a structurally consistent set of Reentrancy=1 contracts.
PLAN-3A was limited by label noise, not architecture.

### H2 — DoS will show non-zero F1 for the first time
**Basis:** dos_loss_weight=0.5 enables gradient on the DoS column with 243
training positives. All prior runs had DoS F1 near 0 (0.019–0.030) — the
model had no learning signal. Even a small F1 gain confirms the weight was
the bottleneck, not insufficient data.

### H3 — Timestamp will hold or slightly improve
**Basis:** 568 structural impossibilities removed. Remaining noise is semantic
(safe uses of block.timestamp) — the model should have less noise to fight.
Risk: 379 remaining Timestamp=1 contracts may still contain 20–35% semantic
mislabeling, limiting how much the model can improve without active learning.

### H4 — ExternalBug will improve
**Basis:** 445 mislabeled contracts removed. ExternalBug previously had 14.8%
structural impossibilities — these were the hardest negative anchors to remove.

### H5 — Overall F1-macro will break through the 0.287–0.288 ceiling
**Basis:** if the ceiling was data-quality-limited, removing 4,304 false positives
should allow the model to learn cleaner patterns and push macro F1 above 0.29.
This is the key test of the data-quality hypothesis.

## Next Steps (after results)

### If H5 confirmed (F1-macro > 0.288)
Data quality is the lever. Continue:
- **Active learning (DQ-5):** surface val contracts with Timestamp prob 0.35–0.65
  for manual review. ~200 corrected labels expected to yield +0.03–0.05 Timestamp F1.
- **Clean negative contracts (DQ-4):** add 100+ audited-clean OpenZeppelin contracts
  with all-zero labels. Fixes safe-contract false-positive problem (currently 0/3
  clean contracts score clean in behavioral testing).
- **PLAN-3D (JK cat mode):** switch `gnn_jk_mode` from `"attention"` to `"cat"`
  after label quality is confirmed improving. Cannot collapse by construction.

### If H5 refuted (F1-macro stays ≤ 0.288)
The ceiling is not label-quality-limited — investigate:
- Are the 4,304 removed labels in val/test (not train)? Check per-split removal counts.
- Is the DoS gradient at 0.5 hurting other classes? Run with dos_loss_weight=0.0 to isolate.
- Run PLAN-3B (CF + DEF_USE only) to complete the ablation matrix.

### If H2 refuted (DoS still near 0 F1)
- Check if DoS positives are all in train or leaking into val
- Try dos_loss_weight=1.0 (full gradient)
- Check DoS augmented contract quality — if synthetic contracts are too unlike real ones,
  the model may not generalise
