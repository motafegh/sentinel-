### Fix 3: Strip Solidifi `bug_*` prefix

**File to create:** `data_module/sentinel_data/preprocessing/solidifi_normalizer.py`

```python
"""Solidifi pre-processor: strip `bug_<type>_*` prefix from function/variable names.

Solidifi synthetic dataset uses function names like:
  - bug_intou4    → Integer Underflow
  - bug_unchk31    → Unchecked block
  - bug_re_ent13   → Reentrancy
  - bug_tmstmp13   → Timestamp dependence
  - bug_txorigin36 → tx.origin
  - bug_unchk_send31 → Mishandled exception

These literal "vulnerability type in name" patterns would let the model learn a
shortcut instead of learning the actual vulnerability patterns. Strip them.

Patterns (from BCCC audit + SolidiFI paper):
  - bug_intou*       (IntegerUO)
  - bug_unchk*       (IntegerUO via unchecked block)
  - bug_re_ent*      (Reentrancy)
  - bug_tmstmp*      (Timestamp)
  - bug_txorigin*    (TransactionOrderDependence)
  - bug_unchk_send*  (MishandledException)
  - bug_dos*         (DoS, fewer instances)
  - bug_leak*        (UnusedReturn)

Replacement: bug_intou4 → f_intou (renamed, no semantic clue).
"""

import re

# Match: bug_<vuln_type><variant_id>  →  f_<vuln_type>
# Examples:
#   bug_intou4      → f_intou
#   bug_unchk31     → f_unchk
#   bug_re_ent13    → f_re_ent
#   bug_tmstmp13    → f_tmstmp
BUG_PATTERN = re.compile(
    r'\bbug_(intou|unchk|re_ent|tmstmp|txorigin|unchk_send|dos|leak|leaky|reentrancy|overflow|underflow)\w*\b',
    re.IGNORECASE
)


def strip_bug_prefix(source: str) -> str:
    """Replace `bug_<vuln_type><id>` with `f_<vuln_type>` to prevent label leakage."""
    return BUG_PATTERN.sub(r'f_\1', source)


# Also: in SolidiFI raw files, the original `.sol` files have comments like
#   // <yes> <report> REENTRANCY
#   // <no> <report> integer_overflow
# These are also removed by `_strip_comments` in the tokenizer, so no change needed.
```

**Where to apply:** `data_module/sentinel_data/preprocessing/normalizer.py` (existing file) — add to the normalization pipeline.

**Validation:**
```bash
# Find a Solidifi contract with bug_intou4
grep -l "bug_intou" data_module/data/preprocessed/solidifi/ | head -1
# Apply normalizer, check that "bug_intou" is gone
ml/.venv/bin/python -c "
from data_module.sentinel_data.preprocessing.solidifi_normalizer import strip_bug_prefix
content = open('data_module/data/preprocessed/solidifi/00683eb3e2b97693a42e8bd1c6cfad32b82ac4eebb0f400a7a86e5ef9f24f821.sol').read()
print('BEFORE:', 'bug_intou' in content)
print('AFTER:', 'bug_intou' in strip_bug_prefix(content))
"
# BEFORE: True
# AFTER: False
```

**Re-preprocess:** `data_module/data/preprocessed/solidifi/*.sol` need re-normalization. ~5 min for 283 files.

**Re-tokenize:** After preprocessing, re-run the v3 export pipeline for SolidiFI portion. (The .pt files for tokens would be regenerated.)

**Effort:** 1-2 hours (including re-preprocess + re-tokenize)
**Risk:** Medium (we're modifying a large chunk of data, need to re-validate)
**Rollback:** Revert the normalizer, restore original preprocessed files from `data/_backup_pre_dos_patch_2026-06-13/` (or new backup)
