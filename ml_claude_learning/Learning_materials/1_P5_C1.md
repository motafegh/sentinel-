 × phase3_emb
                 (where Σαᵢ = 1, learned via softmax)
```

Why? Phase 1 captures *pure structural signal* before it gets oversmoothed by Phases 2+3. JK lets the network route different nodes to different phases — some nodes might rely purely on structure (modifiers, state variables), others on execution order (vulnerable operations).

**Per-Phase LayerNorm — Phase 1-A2:**
Phase 1 runs two conv layers (depth 2), while Phases 2 and 3 also run three conv each. After two convolutions, embeddings can have higher magnitude (L2 norm) than after three. This causes the JK softmax to be *dominated by Phase 1's magnitude* rather than learned phase routing. Solution: **apply LayerNorm once after each phase** before collecting for JK. This normalizes the magnitude across phases, letting attention learn actual routing rather than magnitude artifacts.

### **Summary: 3 Things to Lock In**

1. **11 edge types encode different graph semantics** — Structural (0–5, always on-disk), Directional (6–10, on-disk + runtime). The GNN uses different subsets in each phase because different phases solve different problems.

2. **Three-phase design:** Phase 1 propagates structure downward (function→CFG), Phase 2 captures execution order (CFG paths), Phase 3 propagates order back up to functions then back down to CFG. This is symmetrical: functions now influence their children's embeddings, children's execution patterns now influence functions.

3. **JK + LayerNorm solve oversmoothing:** Throwing away Phases 1 and 2 would lose important signals. JK lets the model blend all three phases with learned weights. LayerNorm ensures magnitude doesn't confound that learning.
